	[목차]
		1 모델 배포 파일 명명 규칙 
		2 모델 배포 파일 Contents 구성

	1 모델 배포 파일 네이밍 
		{MODEL_DEPLOY_HOME}/{INTERFACE_ID}_{MODEL_ID}_{STAGE}.xml
			* {MODEL_DEPLOY_HOME}	: 배포파일 저장 위치 변수 
			* {INTERFACE_ID}		: 인터페이스 ID
			* {MODEL_ID}			: 모델 ID
			* {STAGE}				: 개발(D)/검증(T)/운영(P) 구분 값
			
			
	2 모델 배포 파일 Contents 구성
		2.1 인터페이스 모델 배포 파일 형식 정의서
			아래는 인터페이스 모델 정의서에 적용된 xml 정의서와 설명문 리스트이다.
			1) model.xsd , 인터페이스 모델 정의서  
			2) map.xsd   , 메시지 매핑 정의서 
			3) readme.md , 설명문 
		2.2 인터페이스 모델 배포 파일 Contents 설명
		
			인터페이스 모델 배포 문서 구조에 대한 정의서는 model.xsd 를 참고하라.
					http://idc.mocomsys.com:38080/xsd/model.xsd
					
			2.2.1 형식 
				xml, version 1.0 , encoding UTF-8
			2.2.2 문서 최상위 엘리먼트 
				model
			2.2.3 문서 구조
				model ------------------- 1) 모델 정보  
					/apps --------------- 2) 모델을 실행하는 애플리케이션 정보 
					/layouts ------------ 3) 모델 실행에 필요한 메시지 레이아웃 정보 
					/mappings ----------- 4) 모델 실행에 필요한 매핑 정보 
				  ("/" : child element 표기 , "+" : attribute 표기)  
			2.2.4 모델 배포 문서 내용 및 필드 설명
				1) model
					/id ----------------  5) 모델 ID
					/name --------------  6) 모델 명 
					/stage -------------  7) 실행 스테이지 정보, 개발(D)/검증(T)/운영(P)
					/createDate --------  8) 모델 배포 파일 생성일시 
					/description -------  9) 모델 설명  
					/interface --------- 10) 인터페이스 정보, 복합유형  
						+id ------------ 11) 인터페이스 ID
						+name ---------- 12) 인터페이스 명 
				2) apps ---------------- 13) 애플리케이션 리스트 정보 
					/app --------------- 14) 애플리케이션 정보 
						+id ------------ 15) 애플리케이션 ID
						+name ---------- 16) 애플리케이션 명 
						/type ---------- 17) 애플리케이션 유형 
							+cd -------- 18) 애플리케이션 유형 코드 값 
							+name ------ 19) 애플리케이션 유형 코드 명 
						/system -------- 20) 시스템 정보(애플리케이션이 실행되는) 
							+cd -------- 21) 시스템 코드 값 
							+name ------ 22) 시스템 명 
							+seq ------- 23) 순서(모델 내에서의)
							+type ------ 24) 시스템 유형, 송신(0)/수신(2)
						/server -------- 25) 서버 정보(애플리케이션이 실행되는)
							+cd -------- 26) 서버 코드 값 
							+name ------ 27) 서버 명
						/properties ---- 28) 프로퍼티 리스트 
							/property -- 29) 프로퍼티 
								+name -- 30) 프로퍼티 명 
						/layouts ------- 31) 메시지 레이아웃 리스트(애플리케이션 실행 시 참조될) 
							/layout ---- 32) 메시지 레이아웃 
								+ref --- 33) 참조 메시지 레이아웃 ID (같은 문서 내 "model/layouts/layout" 의 id 값을 가르킴)
								+type -- 34) 메시지 레이아웃 유형, 송신(0)/수신(2)
				3) layouts ------------- 35) 메시지 레이아웃 리스트 
						/layout -------- 36) 메시지 레이아웃, 
						                     CDATA 내에 구체적인 메시지 레이아웃 정의서(XML Schema Definition)가 포함됨 
						                     "2.2.5 메시지 레이아웃" 참고  
							+id -------- 36) 메시지 레이아웃 ID
							+name ------ 37) 메시지 레이아웃 명 
				4) mappings ------------ 38) 매핑 리스트 
						/mapping ------- 39) 매핑,
						                     CDATA 내에 구체적인 매핑 정의 내용이 XML 포멧으로 포함됨
						                     "2.2.6 매핑" 참고  
							+id -------- 40) 매핑 ID
							+name ------ 41) 매핑 명 
			2.2.5 메시지 레이아웃
				메시지 레이아웃의 id 와 name 값은 애플리케이션 및 사람이 레이아웃을 식별할 목적으로 부여된 값이다.
				사용자가 정의한 필드들로 구성된 메시지 레이아웃을 표준 형식인 XSD(XML Schema Definition) 표기에 따라 작성된다.
				XSD 표준 표기법은 아래 사이트를 참고하길 바란다.
				
					https://www.w3.org/2001/XMLSchema
				
			2.2.6 매핑 
				소스 메시지 레이아웃 과 타겟 메시지 레이아웃 간 필드 매핑을 정의한 문서이다.
				매핑문서 정의서는 map.xsd 를 참고하라.
					
					http://idc.mocomsys.com:38080/xsd/map.xsd  
				
				2.2.6 인터페이스 모델 mapping 파일 Contents 설명
					2.2.6.1 형식 
						xml, version 1.0 , encoding UTF-8
					2.2.6.2 문서 최상위 엘리먼트 
						map
					2.2.6.3 문서 구조 및 필드 설명
						("/" : child element 표기 , "+" : attribute 표기)   
						map ---------------------------  1) 매핑 정의서 최상위 엘리먼트 
							/description --------------  2) 매핑 정의서 설명 
								+name -----------------  3) 매핑 정의서 명  
								+cd -------------------  4) 매핑 정의서 코드 값  
							/dataset ------------------  5) 소스 및 타겟 메시지 레이아웃 
								/soruce ---------------  6) 소스 메시지 레이아웃 
									+id ---------------  7) 메시지 레이아웃 ID
								/target ---------------  8) 타겟 메시지 레이아웃
									+id ---------------  9) 메시지 레이아웃 ID 
							/mapping ------------------ 10) 필드 매핑 정보 리스트  
								/item ----------------- 11) 필드 매핑 정보 
									/control ---------- 12) 필드 매핑 컨트롤 정보, "2.2.6.4 매핑 컨트롤 정의" 항목 참고
										+id ----------- 13) 필드 매핑 컨트롤 ID
									/input ------------ 14) 입력 필드 리스트 
										/field -------- 15) 입력 필드 
											+dataset -- 16) 필드가 포함되어 있는 메시지 레이아웃 ID 
											+path ----- 17) 필드의 xpath 값 
									/output ----------- 18) 출력 필드 리스트 
										/field -------- 19) 출력 필드 
											+dataset -- 20) 필드가 포함되어 있는 메시지 레이아웃 ID 
											+path ----- 21) 필드의 xpath 값
					2.2.6.4 매핑 컨트롤
						1) 입력 필드와 출력 필드 간 필드 매핑 시 처리 방법을 식별하기 위해 부여한 코드 값을 말한다.
						   예를 들어 별도의 처리 없이 입력 필드를 출력 필드로 그대로 내보낼때 "move" 컨트롤을 사용하며
						   이 때 사용되는 "move" 컨트롤의 ID 값은 "move"로 표기한다.
						2) 매핑 컨트롤은 필요에 따라 추가 가능하다.
						3) 매핑 컨트롤의 구현 및 런타임 적용에 대한 모든 역할과 의무 및 책임은 런타임 애플리케이션 제품 제공자 또는 사용자에게 있다.
						4) 매핑 컨트롤 ID 리스트 
							move
							substring
							concat
							trim
						
						
	3 외부 연계 방식 
		3.1 서비스 연계
			3.1.1 Provider
				1) 모델배포 서비스를 RESTful 서비스 형태로 제공한다.	
				2) URL
					http://{IIP_SERVICE_URL}/mint/router?method=GET&restId=REST-API-12324895834321491234-57
				3) 파라메터
					{"interfaceMid" : "1"} //인터페이스 모델ID
					
			3.1.2 Client
				1) 모델배포를 원하는 시스템에서 제공한 RESTful 서비스를 호출하여 모델 배포 Contents를 전송한다.    
				2) URL
					서비스 제공자가 지정한 URL
				3) 파라메터 
					서비스 제공자가 지정한 명명 규칙에 따른다. 
		3.2 File 연계
			1) 인터페이스모델 관리 시스템이 실행되고 있는 로컬 파일 시스템(로컬 인식이 가능한 공유 파일 시스템 포함)에 
			   모델 배포 Contents를 파일로 저장한다.
			2) 파일이 저장된 이후 모델파일을 참조하길 원하는 원격지 시스템으로의 연계 처리에는 관여하지 않는다.
			3) 모델 배포파일 네이밍
				{MODEL_DEPLOY_HOME}/{INTERFACE_ID}_{MODEL_ID}_{STAGE}.xml
		3.3 형상관리 연계
			1) 인터페이스 모델관리 시스템이 구축될 사이트의 형상관리 시스템에서 제공하는 연계방식에 맞게 인하우스 개발한다.
			2) 형상관리 인하우스 개발을 추사화하는 인터페이스 계층을 디자인한다.
